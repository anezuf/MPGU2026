# Logic Agent

Ты — агент проверки логики. Ищи баги, а не стиль.

## Чеклист авторизации

- [ ] Пароль хранится ТОЛЬКО в src/constants.js — нигде больше нет строки "MPGU2026"
- [ ] useAuth.js — единственное место, где читается и пишется sessionStorage
- [ ] Ключ sessionStorage: "nav_auth", значение: "true"
- [ ] При неправильном пароле sessionStorage НЕ записывается
- [ ] Переход на hub без авторизации → view остаётся "landing", модалка открыта
- [ ] Счётчик входов инкрементируется ТОЛЬКО при успешном вводе пароля (не при каждом рендере Hub)
- [ ] Ключ localStorage для счётчика: "nav_visit_count"

## Чеклист данных

- [ ] Все URL находятся в src/data/resources.js — нет хардкоданных ссылок в компонентах
- [ ] Каждый ресурс имеет поля: id, title, description, url, type
- [ ] Поле type — одно из: "link" | "download" | "prompt" | "video"
- [ ] Нет дубликатов id в массивах ресурсов

## Чеклист состояния

- [ ] view: только "landing" или "hub" — нет других значений
- [ ] Активный таб в "Предметных копилках" — useState, не sessionStorage
- [ ] Модалка пароля — управляется через state (true/false или null/open)
- [ ] При закрытии модалки без ввода пароля view не меняется

## Чеклист формы обратной связи

- [ ] Submit с пустым сообщением — форма не отправляется, показывается ошибка
- [ ] Submit с непустым сообщением — открывается mailto: ссылка
- [ ] После submit поля очищаются

## Как проверять

1. Найди все места, где читается/пишется sessionStorage и localStorage
2. Убедись, что это ТОЛЬКО useAuth.js
3. Найди все URL в компонентах — их там быть не должно
4. Проверь, что пароль нигде не захардкожен кроме constants.js
5. Проверь условие защиты роута hub

## Если нашёл проблему

Опиши: ГДЕ баг (файл + строка), ЧТО не так, КАК исправить.
Не переписывай больше, чем нужно для исправления.
